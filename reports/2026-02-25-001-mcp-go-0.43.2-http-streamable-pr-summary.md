## Executive Summary

This change set upgrades `github.com/mark3labs/mcp-go` from `v0.17.0` to `v0.43.2`, migrates all MCP tool handlers to `request.GetArguments()`, and introduces Streamable HTTP transport as an opt-in alternative to stdio via `--transport` / `SAP_TRANSPORT`.

The implementation is backward compatible because `stdio` remains the default transport.

## Problem Statement

The previous implementation had three gaps:

1. It used an older MCP SDK version (`v0.17.0`) that does not reflect the newer Streamable HTTP server flow.
2. Tool handlers still relied on direct access through `request.Params.Arguments`, while the migration target requires `request.GetArguments()`.
3. Runtime transport was fixed to stdio only, with no HTTP Streamable option.

## Implemented Changes

### 1) Dependency Upgrade

- Updated `go.mod`:
  - `github.com/mark3labs/mcp-go v0.17.0` -> `v0.43.2`
- Updated `go.sum` with required transitive dependencies.

### 2) Handler Migration to `GetArguments()`

- Migrated all MCP tool handlers from:
  - `request.Params.Arguments[...]`
- To:
  - `request.GetArguments()[...]`
- Migration was applied across all `internal/mcp/handlers_*.go` files.

### 3) New Transport Option: `http-streamable`

- Added CLI flag in `cmd/vsp/main.go`:
  - `--transport` (`stdio` default, `http-streamable` optional)
- Added env var support:
  - `SAP_TRANSPORT`
- Added config normalization and validation for transport values.
- Added server routing methods in `internal/mcp/server.go`:
  - `Serve(transport string)`
  - `ServeStdio()`
  - `ServeStreamableHTTP(addr string)`
- Streamable HTTP defaults:
  - Address: `127.0.0.1:8080`
  - Endpoint: `/mcp`

### 4) Documentation Alignment

- Updated docs to reflect dual transport support and new configuration entry:
  - `ARCHITECTURE.md`
  - `CLAUDE.md`
  - `README.md`

### 5) Build Compatibility Context (`GetDependencyZIP`)

- Kept `GetDependencyZIP(...)` available in `embedded/deps/embed.go` as minimal compatibility behavior (returns `nil` in current embedded state), aligned with current upstream direction while PR #31 is still open.
